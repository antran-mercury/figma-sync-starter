### Trigger

User **clicks / taps** a Hotspot marker on the image.

### Behavior

- Hotspots are circular markers placed at specific coordinates on the image.
- A subtle **pulse animation** is applied to each hotspot in the default state to indicate interactivity.
- When a hotspot is clicked, a **Tooltip** appears relative to that marker's position, showing:
  - Product Name
  - Price
  - CTA (call-to-action button)
- **Single-active rule**: opening a new tooltip automatically dismisses the currently active one — only one tooltip can be open at a time.
- The clicked hotspot enters an **Active state** (color change or icon swap) to highlight the selected item.

### Dismissal

The tooltip closes when the user:
- Clicks the same hotspot again (toggle off).
- Clicks a different hotspot (which then opens its own tooltip).
- Clicks anywhere outside the tooltip area (click-outside to dismiss).

### States

- `default`: Hotspot visible with pulse animation, no tooltip open.
- `active`: Hotspot is highlighted (color/icon change), tooltip is visible.

### Animation / Transition

- Pulse animation on hotspot in default state (loop, subtle scale or opacity pulse).
- Tooltip open/close transition — duration and easing to be defined by implementation.

### Notes for AI

- Only one tooltip may be active at a time; enforce the single-active rule in state management.
- Tooltip position is relative to the hotspot marker — handle viewport overflow by flipping direction if needed.
- Click-outside detection must not interfere with clicking other hotspots.
- z-index must be high enough so the tooltip is not covered by other elements on the image.

#### Trigger

User hovers over the hotspot icon on the map.

#### Behavior

- Show a tooltip popup above the icon
- Tooltip fades in with a duration of 200ms
- Keep the tooltip visible when the user moves the cursor into the tooltip itself
- Hide the tooltip when clicking outside or moving the cursor away

#### States

- `default`: Icon is displayed normally, no tooltip
- `hover`: Tooltip appears, icon is highlighted
- `active`: Tooltip is pinned (click the icon to pin/unpin)

#### Animation / Transition

- Fade in: `opacity` 0 → 1, duration 200ms, easing `ease-out`
- Fade out: `opacity` 1 → 0, duration 150ms, easing `ease-in`

#### Notes for AI

- Tooltip must not overflow the viewport — if there is not enough space above, automatically flip it below the icon
- z-index must be high enough so the tooltip is not covered by other elements on the map
